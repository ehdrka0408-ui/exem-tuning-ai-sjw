# 화면별 기획 결정 사항

## V2 작업 파이프라인 (V2WorkPipeline)

### 이 화면의 목적 (우선순위순)

1. **일괄 검증 처리** — 매일 아침, 야간 자동튜닝 결과를 빠르게 검증완료 처리. 기본이 전체 처리, 예외만 빼는 흐름. 트리아지(개선률 낮은 건 걸러내기) 포함.
2. **현황 조감** — 전체 작업이 어떤 상태에 있는지 한눈에 파악. 칸반이 이 역할.
3. **내 요청 추적** — "내가 어제 넣은 그 배치 어떻게 됐지?" 단건 또는 배치 묶음 단위로 찾기. 빈도는 낮지만 못 찾으면 체감 불편 큼.

### 뷰 설계 방향

- **칸반 뷰**: 상태별 조감 + tuned 컬럼 일괄선택(체크박스). 조감 역할 유지하면서 일괄검증도 가능.
- **리스트 뷰**: 일괄 검증 처리 전용. 체크박스 + 플로팅 바, 개선률 정렬(트리아지), 인라인 빠른 액션(검증완료/튜닝불가).
- **요청단위별 칸반**: 기각. 요청이 100개 넘으면 컬럼 무한 증가 → 조감 불가. 목적 3(내 요청 추적)은 "내 작업만" 프리셋 필터로 해결.

### 일괄검증 구현 결정

- 칸반/리스트 모두 일괄검증 지원 (체크박스 + 하단 플로팅 바 공유)
- 칸반: tuned 컬럼 헤더에 전체선택, 카드 우측 상단에 개별 체크박스
- 리스트: tuned 행에만 체크박스, 인라인 빠른 액션(검증완료/튜닝불가), tuned 우선 정렬 + 개선률 내림차순

## 튜닝 상태 흐름 및 재튜닝 루프 정책

### 두 개의 독립 축

| 축 | 값 | 설명 |
|---|---|---|
| **인덱스 포함 여부** | 실측 (일반 SQL) | 인덱스 생성 없이 실행계획 확인 가능 |
| | 예상 (인덱스 포함) | 인덱스 생성해야 실측 실행계획 확인 가능 → 비인덱스 대안도 제공 필수 |
| **타임아웃 여부** | 정합성 검증 완료 | 튜닝 전/후 SQL 모두 정상 실행 완료 |
| | 미검증 (타임아웃) | 튜닝 전/후 SQL 중 하나 이상이 10분 타임아웃 |

- 타임아웃 = SQL 실행 타임아웃 (튜닝 전 10분, 튜닝 후 10분)
- MINUS 쿼리(정합성 검증)도 별도로 타임아웃 가능

### 재튜닝 루프 정책

- **내부 재시도**: 사용자에게 노출하지 않음. 최종 best 결과만 전달.
- **개선율 기준**: 70% (상사 결정). 사용자 개인 의견은 1%도 가치 있다고 봄.
- **룰 엔진**: 적극 테스트 예정. 룰 엔진은 동일 입력 → 동일 출력이므로 재시도 의미 없을 수 있음 (프롬프팅 변경 시에만 의미).

### 재튜닝 루프 트리거 매트릭스 (6 케이스)

| # | 튜닝 전 실행 | 튜닝 후 실행 | 정합성 검증 | 루프 트리거? | 비고 |
|---|---|---|---|---|---|
| 1 | 완료 | 완료 | 완료 | 개선율 < 70%이면 재시도 | **결정됨** |
| 2 | 완료 | 타임아웃 | — | 재시도 | **결정됨** — 튜닝 후가 더 느려진 경우 |
| 3 | 타임아웃 | 완료 | — | **미결** | 개선은 됐는데 개선율 계산 불가 |
| 4 | 타임아웃 | 타임아웃 | — | **미결** | 둘 다 타임아웃, 비교 불가 |
| 5 | 완료 | 완료 | 타임아웃 | **미결** | SQL은 되는데 MINUS가 타임아웃 |
| 6 | 완료 | 완료 | 실패 (불일치) | **미결** | 결과 불일치 — 정합성 깨짐 |

### 타임아웃 시 프롬프팅 분기 가능성

- 타임아웃 케이스에서 프롬프트를 다르게 구성하여 재시도 가능 (예: "이 SQL은 10분 내 실행 불가했으므로 성능 최적화에 더 집중해줘")
- 이 부분도 미결 — 룰 엔진 테스트 결과에 따라 결정

## 튜닝안 구성 규칙

- **인덱스안이 없는 경우**: 튜닝안 1개 (힌트/리라이트 등)
- **인덱스안이 있는 경우**: 반드시 2개 — "인덱스안" + "차선안"(인덱스 없이 가능한 최선)
- 복수 안이 생기는 유일한 이유는 인덱스 포함 여부임
- 라벨: "인덱스안" / "차선안" (기존 "튜닝안 A/B" 폐기)

## 작업상세 UX 구조

### 슬라이드 패널 (WorkPipeline에서 행 클릭)
- 목록 컨텍스트 유지하며 우측 40% 패널로 상세 확인
- 정보 순서: 요약 헤더 → 튜닝안 선택(복수 시) → 메트릭 요약 → 전/후 탭(SQL+실행계획) → 전체 비교 → AI 근거
- 전/후 탭: `[전 (AS-IS)] [후 (TO-BE)]` 순서, 기본 "후" 선택
- 매 건 SQL+실행계획 반드시 확인하는 전제 → 아코디언 접기 없이 바로 노출
- 새 탭(↗ 버튼): 풀스크린 WorkDetail을 새 브라우저 탭으로 (`?popup=true` → 메뉴바 숨김)
