# 작업 상세 — 권고안

## 화면 목적

승인 완료된 튜닝안의 DDL 구문과 SQL 변경 내용을 개발자·DBA에게 전달하기 위한 조회·복사·다운로드 화면. 인덱스 DDL과 SQL 변경을 한 화면에서 확인하고 클립보드 복사 또는 .sql 파일로 내보낼 수 있다.

---

## 레이아웃

검토 탭과 동일한 슬라이드 패널 내부에서 표시 (→ spec-work-detail-review.md 레이아웃 참조).

---

## 탭 목적

승인 완료된 튜닝안의 인덱스 DDL 구문과 SQL 변경 .sql 파일을 복사·다운로드하여 개발팀에 전달하는 화면.

### 사용자 시나리오

**시나리오 1 — DDL + SQL 권고안 전달**
튜너가 승인 완료 후 권고안 탭 진입 → 인덱스 DDL 복사 → .sql 파일 다운로드하여 개발팀에 전달

**시나리오 2 — 인덱스 DDL 확인**
DBA가 승인 완료 건의 권고안 탭에서 인덱스 DDL 구문 확인 → 복사 버튼으로 클립보드에 복사 → 별도 세션에서 수동 실행

---

## Description

### (1) 인덱스 DDL

**1. 내용**
- 튜닝안에 인덱스 생성이 포함된 경우, 각 인덱스의 DDL 구문을 보여준다.

**2. 구성**
- a. 인덱스 카드 (N개)
 - 인덱스명 — 고정폭 파란 글꼴
 - 복사 버튼 — DDL 클립보드 복사
 - DDL 코드 — 하단 고정폭 표시, SQL 키워드 하이라이트

**3. 동작**
- a. 복사 클릭 → 클립보드 복사, "복사됨 ✓" 피드백 1.5초

---

### (2) SQL 변경 (Hint / Rewrite)

**1. 내용**
- 튜닝안에 SQL 변경(힌트 추가 또는 리라이트)이 포함된 경우, .sql 파일 형태로 전체 내용을 보여준다.

**2. 구성**
- a. 섹션 제목 — "SQL 변경" + 유형 표시 (Hint / Rewrite / Hint + Rewrite)
- b. SQL 카드
 - 파일명 — `{작업ID}-recommend.sql`
 - .sql 전체 내용 (pre 영역, max-h 400px, 스크롤)
 - 헤더 주석: 작업 ID, SQL ID, 인스턴스, 스키마, 튜닝안, 생성일
 - 성능 비교: Elapsed Time / Buffer Gets Before→After (개선율)
 - 변경사항 요약 + 근거 (최대 5줄)
 - 선행 조건 경고 (인덱스 포함 시)
 - 바인드 변수 목록
 - 튜닝 전 SQL + 실행계획(Before)
 - 튜닝 후 SQL + 실행계획(After)
 - 면책 안내문
 - 구문 강조: 섹션 제목=파랑, 경고=빨강, 복사 안내=초록, 구분선=연회색, 주석=회색, SQL 키워드=blue
- c. .sql 파일 템플릿 예시
```sql
-- ============================================================
-- SQL 튜닝 권고사항
-- ============================================================
-- 작업 ID : WI-2024-001
-- SQL ID : abc123def456
-- 인스턴스 : PROD_DB_01
-- 스키마 : HR
-- 튜닝안 : 안 1 — 인덱스 + 힌트
-- 생성일 : 2026-04-13
--
-- [성능 비교]
-- Elapsed Time : 2,340ms → 120ms (-94.9%)
-- Buffer Gets : 45,230 → 1,820 (-96.0%)
--
-- [변경사항]
-- EMP_DEPT_IDX 인덱스 추가 + INDEX 힌트 적용으로 Full Table Scan 제거
--
-- [근거]
-- 1. EMP 테이블 DEPT_ID 컬럼 카디널리티 분석 — 선택도 0.3%
-- 2. Nested Loop Join 유도로 Buffer Gets 대폭 감소
--
-- ⚠ 선행 조건: 아래 인덱스 생성 완료 후 적용하세요
-- - IDX_EMP_DEPT_ID
--
-- [바인드 변수]
-- :dept_id = '100'
-- :status = 'ACTIVE'
-- ============================================================

-- [튜닝 전 SQL]
SELECT * FROM emp WHERE dept_id = :dept_id AND status = :status;

-- [실행계획 — Before]
---------------------------------------------
| Id | Operation | Name | Rows | Cost |
---------------------------------------------
| 0 | SELECT STATEMENT | | 500 | 1234 |
| 1 | TABLE ACCESS FULL| EMP | 500 | 1234 |
---------------------------------------------

-- ------------------------------------------------------------
-- [튜닝 후 SQL] 힌트 변경
-- ▼ 복사하여 사용
-- ------------------------------------------------------------

SELECT /*+ INDEX(emp IDX_EMP_DEPT_ID) */ * FROM emp WHERE dept_id = :dept_id AND status = :status;

-- [실행계획 — After]
-----------------------------------------------------------
| Id | Operation | Name | Rows | Cost |
-----------------------------------------------------------
| 0 | SELECT STATEMENT | | 2 | 5 |
| 1 | TABLE ACCESS BY INDEX ROWID| EMP | 2 | 5 |
| 2 | INDEX RANGE SCAN | IDX_EMP_DEPT_ID | 2 | 3 |
-----------------------------------------------------------

-- ============================================================
-- 본 권고는 AI 튜닝 시스템에서 자동 생성되었습니다.
-- 적용 전 반드시 개발/DBA 검토 후 진행하세요.
-- ============================================================
```
- d. 액션 바 (하단)
 - 전체 복사 버튼
 - .sql 다운로드 버튼

**3. 동작**
- a. 전체 복사 클릭 → 클립보드 복사, "복사됨 ✓" 피드백 2초
- b. .sql 다운로드 클릭 → `{작업ID}-recommend.sql` 파일 다운로드 (UTF-8). 첫 다운로드 시 이력에 `sql_downloaded` 이벤트 추가

---

### (3) 인덱스만 있는 경우

**1. 내용**
- SQL 변경 없이 인덱스 생성만으로 개선되는 경우 안내 메시지를 표시한다.

**2. 구성**
- a. 안내 텍스트 — "SQL 변경 없이 인덱스 생성만으로 개선됩니다."

**3. 동작**
- 없음 (읽기 전용)

---

### 조건분기

- 인덱스 포함 튜닝안 → (1) 인덱스 DDL + (2) SQL 변경 표시 *(4/23 변경: 1건 1안)*
- 인덱스 미포함 튜닝안 → (2) SQL 변경만 표시
- 인덱스만 포함 (SQL 변경 없음) → (1) + (3) 표시
