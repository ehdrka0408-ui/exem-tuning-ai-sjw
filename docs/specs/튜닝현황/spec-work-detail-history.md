# 작업 상세 — 이력

## 탭 목적

해당 튜닝 건의 전체 생애주기를 시간 역순 타임라인으로 보여주는 탭. 작업 생성부터 현재 상태까지 누가 언제 무엇을 했는지 추적한다.

> **현재 구현 상태** *(2026-04-24 기준)*: 슬라이드 패널 / 워크스페이스의 탭 바에 **이력 탭 UI 자체가 미구현**. 본 명세의 모든 내용은 구현 예정 기획. 상태 전이·rationale 변경 이력은 백엔드 `tuning_requests.status` / `rationale` / `completed_at` 값으로 부분 추적 가능하나 별도 이력 테이블은 없음 — 구현 시 이력 테이블 신규 추가 필요.

### 사용자 시나리오

**시나리오 1 — 반려 사유 확인**
튜너가 반려된 건의 이력 탭 진입 → 타임라인에서 반려 이벤트 확인 → 반려 사유 블록에서 상세 사유 읽기

**시나리오 2 — 재튜닝 경위 파악**
재튜닝 요청된 건의 이력 탭 → 1차 튜닝완료 → 재튜닝 요청(사유 노트) → 2차 튜닝 시작/완료 흐름을 시간순으로 파악

## Description

### (1) 타임라인

**1. 내용**
- 해당 작업의 상태 변경 이력을 시간 역순(최신 먼저) 타임라인으로 보여준다.

**2. 구성**
- a. 타임라인 세로선 — 좌측 1px, 이벤트 도트 연결
- b. 이벤트 항목 (N개, 시간 역순)
 - 도트 — 이벤트 유형별 색상
 - 이벤트 라벨 — 굵은 텍스트
 - 행위자 — 사용자명 또는 "AI Engine" / "System"
 - 타임스탬프 — YYYY-MM-DD HH:MM 형식
 - 노트 — 선택적 부가 설명 (있을 때만 표시)
 - 반려 사유 블록 — 반려 이벤트에만 표시

**3. 동작**
- 없음 (읽기 전용)

---

| 이벤트 | 라벨 | 도트 색상 | 비고 |
|--------|------|----------|------|
| created | 작업 생성 | 회색 (border) | 자동/수동 선정 구분은 노트에 표시 |
| tuning_started | 튜닝 시작 | 회색 (border) | 행위자: AI Engine |
| tuning_completed | 튜닝 완료 | 파랑 (code) | 행위자: AI Engine |
| approved | 승인 | 초록 (success) | 행위자: 튜너 이름 |
| retune_requested | 재튜닝 요청 | 노랑 (warning) | 노트에 재튜닝 사유 |
| rejected | 반려 | 빨강 (danger) | 반려 사유 블록 별도 표시 |
| exported | SQL 전달 | 파랑 (info) | 노트에 수신자 |
| index_created | 인덱스 생성 | 파랑 (info) | 노트에 인덱스명 |

| index_schedule_failed | 예약 실행 실패 | 빨강 (danger) | 예약 실행 실패 시. 행위자: System |
| sql_downloaded | 권고안 다운로드 | 회색 (border) | .sql 첫 다운로드 시만. 노트에 파일명 |

| failed | 실패 | 빨강 (danger) | |
| cancelled | 취소 | 빨강 약화 (danger/60) | |
| no_improve | 개선없음 | 회색 (text-muted) | |

### (3) 반려 사유 블록

**1. 내용**
- 반려 이벤트에 첨부된 사유를 별도 블록으로 표시한다.

**2. 구성**
- a. 블록 컨테이너 — 테두리 + 배경색 구분
- b. 헤더 — "반려 사유"
- c. 사유 텍스트 — 기본 3줄 표시, 초과 시 접기/펼치기
- d. 사유 없음 시 — "사유 없음" 이탤릭

**3. 동작**
- a. 3줄 초과 또는 200자 초과 시 "더 보기" / "접기" 토글 노출

---

### (4) 이력 없음 상태

**1. 내용**
- 이력 데이터가 없는 경우 빈 상태 메시지를 표시한다.

**2. 구성**
- a. 안내 텍스트 — "이력 정보가 없습니다." (중앙 정렬)

**3. 동작**
- 없음 (읽기 전용)
