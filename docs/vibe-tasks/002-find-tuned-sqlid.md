# 과제 002: 튜닝 후 SQL_ID 찾기 + 등록

## 해결하려는 문제

튜닝 SQL이 운영 DB에 반영된 후, V$SQL에서 해당 SQL의 SQL_ID를 찾아야 함.
SQL_ID를 알아야 Before/After 성능 수치 비교 가능 → 운영효과 추적의 전제조건.

## 핵심 전략: 주석 삽입

튜닝 SQL 시작부에 주석 자동 삽입:
```sql
/* AI for tuneID: agvjeklqlsn */ SELECT ...
```

### 전제 조건

| 조건 | 방식 |
|------|------|
| MaxGauge 없음 | DBA_HIST_SQLSTAT / V$SQL 직접 조회 |
| MaxGauge 있음 | DMA(모니터링 부하 최소화) 경로 |

## 매칭 3단계 폴백

1. **주석 정확매칭** — tuneID 주석으로 V$SQL 검색 (최선)
2. **다이나믹 SQL 추측** — 주석 매칭 결과 여러 건이면 동적 SQL 변형 추정
3. **해싱 유사도** — 주석 제거 후 정규화 해시 비교 → 유사도 순 후보

## 권고 콘텐츠 가이드 (사용자에게 노출)

> "운영효과 추적을 위해 아래 주석을 소스에 함께 반영해주세요"
> → 주석 포함 복사 시 운영효과 자동 확인 가능

## UI: SQL_ID 등록 화면

- 좌: 후보 리스트 (SQL_ID + SQL_TEXT 20자)
- 우: 비교 뷰 — 튜닝 후 SQL | 후보 SQL + Diff 하이라이트
- 하단: [이 SQL_ID로 등록] → 운영효과 추적 시작
